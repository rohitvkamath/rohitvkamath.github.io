Content Management

This guide explains how to update the primary content of the portfolio.

Updating Work Experience​

The work experience is managed in src/pages/work.md.

  1. Open src/pages/work.md.
  2. Add a new entry under the ## Work Experience section.
  3. Follow the existing format: 
    #### <div style={{alignItems: 'start', marginTop: 20}}><div style={{float: 'left'}}>Company Name, Location. (Date Range)</div> </div>
        - Description of the company.
        - Key responsibilities and achievements.

Adding Blog Posts​

Blog posts are located in the /blog directory.

  1. Create a new file or folder in /blog.
  2. If using a folder, name it YYYY-MM-DD-slug/index.md.
  3. Include front matter at the top: 
    ---
    slug: my-post-slug
    title: My Post Title
    authors: [rohit]
    tags: [tech, leadership]
    ---

Updating the Home Page​

The home page content is primarily driven by src/components/profile_summary.mdx.

  • Bio/Summary: Edit the text in src/components/profile_summary.mdx.
  • Profile Picture: Replace static/img/me.jpg or update the path in the MDX file.
  • Tagline: Update the tagline in docusaurus.config.js.

AI-Assisted Updates​

If you are using Cline to update content, you can simply say:

  • "Cline, add a new work experience entry for [Company] where I [Responsibilities]."
  • "Cline, create a new blog post about [Topic]."

Cline will follow the patterns defined in this repository and the .clinerules file.
